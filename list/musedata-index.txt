http://bit.ly/musedata-menu								
This spreadsheet controls the file list menu on the musedata website (http://www.musedata.org)								
The SHORTCUT column for each file must be unique and is required.  This value will be used for the downloaded filename, and also to access data on the server in the form:    http://pdf.musedata.org/id?beet-sym1-study-2008								
LEVEL_1 should almost always be the composer's name								
LEVEL_2 could either be a genre, a collection (such as WTC), or a work name.								
The menu system allows up to 5 levels of categorization for the files (level 6 and higher are not allowed)								
The files will be listed in the menu in the order they are placed in this spreadsheet (with the exception that if more Beethoven files are added after another compoer's entries, they will be moved to the end of the Beethoven list).								
The DATA_URL column is the link to the data used to produce the PDF.  This must be the "raw/master" link to the data (not to the webpage where you can edit the data file).  Note in particular that there will be a long alphanumeric code when viewing the raw file on Bitbucket.  This string should be replaced with "master" (which means the most recent version of the file). 								
The URL column is an optional link to the CCARH wiki page related to the given work.
